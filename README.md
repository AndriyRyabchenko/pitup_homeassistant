# PitUp — інтеграція для Home Assistant

Інформер стану вашої техніки з [PitUp](https://pitup.app) у Home Assistant:
скоро / протерміновано ТО, найближче обслуговування, пробіг — окремою сутністю
на кожну техніку.

## Встановлення (HACS)

1. HACS → **Integrations** → меню ⋮ → **Custom repositories**.
2. Додайте репозиторій `https://github.com/AndriyRyabchenko/pitup_homeassistant`,
   категорія **Integration**.
3. Знайдіть **PitUp** у списку та встановіть.
4. Перезапустіть Home Assistant.

### Вручну

Скопіюйте теку `custom_components/pitup` у `config/custom_components/` вашого HA
й перезапустіть.

## Налаштування

1. У PitUp: **Налаштування → Home Assistant → Згенерувати токен**, скопіюйте токен.
2. У Home Assistant: **Налаштування → Пристрої та служби → Додати інтеграцію → PitUp**.
3. Встав **API-токен** (адресу вводити не треба — вона фіксована `https://pitup.app`).

Створяться сутності:

- `sensor.pitup` — загальний стан парку (`ok` / `soon` / `overdue`), а в атрибутах —
  зведення по всій техніці (`vehicles`, `totals`).
- `sensor.pitup_<техніка>` — стан кожної техніки з деталями (пробіг/мотогодини, найближче ТО, пункти).
- `number.<техніка>_лічильник` — лічильник пробігу/мотогодин (можна змінити або писати з автоматизації).

> Яку саме техніку віддавати — обирається галками в кабінеті PitUp (профіль → Home Assistant).

## Картка на дашборд

**Рекомендований формат — Markdown-картка** (`lovelace-card.yaml`). Вона нативна, тож ніколи не
дає «Custom element not found» (на відміну від кастомних карток на холодному кеші). Показує
таблицю: світлофор статусу 🟢🟡🔴, техніка + пробіг/мотогодини, найближче ТО і **попередження про
страховки** (ОСЦПВ, що спливає/прострочене); зверху — підсумок «протерміновано / скоро / страховки».

Як додати: **дашборд → Додати картку → Ручна (Manual)** → встав вміст `lovelace-card.yaml`.

### Бічна панель «PitUp»

Ставиться **разом з інтеграцією** — готовий інформер по всій техніці у бічному меню (теж без
проблем із завантаженням).

### Кастомна картка (опційно)

`custom:pitup-card` доступна у списку **Додати картку**. Зауваж: як і будь-яка кастомна
Lovelace-картка, на першому завантаженні з очищеним кешем можлива помилка «Custom element not
found» (обмеження HA-фронтенду) — зникає після F5. Для стабільності радимо Markdown-картку вище.

## Запис лічильника (пробіг / мотогодини)

Інтеграція вміє **писати** лічильник назад у PitUp:

- **Number-сутність** `number.<техніка>_лічильник` — показує поточне значення; можна змінити вручну
  або з автоматизації.
- **Сервіс** `pitup.set_counter` (`vehicle_id`, `value`) — монотонний (не зменшує).

### Приклад: мотогодини генератора з binary_sensor

Маєш `binary_sensor.deye_generator` (on = працює)? Рахуємо напрацювання і пишемо в PitUp.
Готовий пакет — `examples/generator-motohours.yaml`: template `0/1` → Riemann-інтеграл у години →
`offset + напрацювання` → `pitup.set_counter`. Тоді регламент генератора «кожні N мотогодин»
рахується від реальних годин.

## Сповіщення (нативні пуші)

Замість (чи на додачу до) Telegram — **нативні пуші в застосунку Home Assistant** про нові
«скоро / прострочено» (ТО і страховки). Увімкнути: **Налаштування → Пристрої та служби → PitUp →
Налаштувати (Configure)** → вкажи свій notify-сервіс, напр. `mobile_app_твій_телефон` (порожньо —
вимкнено). Інтеграція сповіщає **лише про нові** попередження (без спаму при кожному опитуванні).

## Опитування

Дані оновлюються раз на годину (`cloud_polling`).
